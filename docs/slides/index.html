Naughty Agents

Human-in-the-loop Security for On-chain AI Agents

ETHGlobal NYC 2025

Problem

  • AI agents can be hijacked or tricked into malicious transactions
  • On-chain actions are irreversible
  • Users need a safety net without blocking good automation

Solution: Naughty Agents Protocol

  • Security Module at wallet layer
  • Web of Trust of human reviewers (stake + invites)
  • Review Oracle for flagging, voting, resolving
  • Action Registry stores blacklisted action hashes

Architecture

  • SCA Security Module enforces Registry checks
  • Review Oracle manages queue, votes, resolution
  • Web of Trust invites, stakes, slashing (delegated)
  • Action Registry on-chain blacklist (bytes32)

Atomic Action Hashing (A1)

Deterministic Keccak-256 over canonical JSON


Action: "native_transfer"
Params: { "to": "0x...0001", "amount": 100 }
Canonical JSON: {"action":"native_transfer","params":{"amount":100,"to":"0x...0001"}}
Keccak-256: 0x806b7287a2da74039e73e3171087ca87ff859a08b743fc6e2a4ef168eefa0c52

Demo Flow

  1. Agent proposes action → compute hash (A1)
  2. Flag to Review Oracle
  3. Reviewers vote (quorum)
  4. Resolve → Action Registry blacklists hash
  5. Security Module blocks blacklisted actions
